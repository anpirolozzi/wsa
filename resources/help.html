Guida rapida di Web Site Analyser

Funzionalità principali

L' applicazione permette di analizzare siti web ed aiutare ad eseguirne la manutezione.

Permette in particolare di eseguire le seguenti operazioni:

  • Analisi di siti web tramite immissione manuale dell'URI dominio
  • Analisi di siti da file tramite selezione grafica del dominio o immissione manuale dell'URI dominio
  • Aggiunta e visualizzazione degli URI Seed in qualsiasi momento una volta aperta l'esplorazione
  • Archiviazione automatica (se prevista dall'utente)
  • Apertura di un archivio precedentemente salvato
  • Mettere in pausa, riprendere o fermare l'esecuzione di un analisi
  • Mostrare lo stato di scaricamento degli URI e visualizzarne informazioni correlate
  • Visualizzare informazioni statistiche su un dominio
  • Calcolare il numero di link da un dominio verso un altro (o verso se stesso)

Creazione e apertura di un analisi

Creazione di un analisi

Per creare una sessione di analisi basta premere sul pulsante Nuova analisi. Nella finestra di dialogo sarà possibile specificare manualmente l' URI dominio o nel caso di dominio locale selezionarlo tramite interfaccia grafica.

Un URI dominio valido per poter essere analizzato deve possedere la seguente sintassi

scheme://authority[path]

La parte path è opzionale, la parte authority consiste solamente dell'host. L'URI dominio che specifica un dominio puo anche non essere una pagina scaricabile

Esempio

http://docs.oracle.com/javase/specs/jls/se8

file:///C:/Cartella/

Se lo si desidera si può attivare la funzione di archiviazione automatica. Spuntare la casella Archivia l'analisi. Tale funzione permettere di archiviare l'analisi automaticamente ogni 50 secondi o ad avvenuta sospensione dell'analisi.

Una volta specificato l'URI dominio siamo pronti per creare la sessione di analisi. Premere Ok. Nel caso è stata attivata l'archiviazione verrà mostrata un ultima finestra per selezionare la directory di destinazione dell'archivio. Adesso siamo pronti per davvero. La sessione di analisi è stata aperta.

Creazione di un analisi tramite interfaccia

Nel caso in cui il dominio desiderato proviene da una fonte locale è possibile generare il suo URI tramite interfaccia. Premere Sfoglia...

Selezionare l'opzione più opportuna a seconda che il dominio desiderato sia un file o una cartella locale

Una volta selezionato, l'URI del dominio apparirà automaticamente nella casella di testo presente nella finestra di dialogo Nuova analisi.

Anche in questo caso è possibile attivare la funzione di archivazione automatica se lo si desidera.

Impostati i parametri possiamo anche in questo caso premere Ok per creare la sessione d'analisi

Apertura di un analisi precedentemente salvata

Nel caso si possiede un archivio di una precedente analisi, è possibile ripristinare la sessione di analisi salvata.

Per fare ciò basta premere il pulsante Apri analisi. Selezionate la cartella contenente l'archivio e confermare. L'analisi salvata verrà quindi ripristinata e automaticamente riprenderà dal punto interrotto se non terminata.

Specifica di URI Seed

Per poter effettuare un'esplorazione del sito web è necessario specificare uno o più indirizzi URI di pagine web che appartengono al dominio, questi sono detti seed.

Esempio

Se il dominio è:

http://twiki.di.uniroma1.it/pub/Metod_prog

un URI Seed potrebbe essere:

http://twiki.di.uniroma1.it/pub/Metod_prog/RS_L18/lezione18.html

Chiaramente gli URI Seed devono corrispondere a pagine web e devono appartenere al dominio del sito web.

Aperta la sessione di analisi è possibile inserire gli URI Seed e visualizzare quelli già inseriti in ogni momento. Premere il bottone Seed.

Similmente all'inserimento del dominio è possibile specificare l'URI Seed manualmente e nel caso di dominio locale tramite un interfaccia. Una volta comparso l'URI Seed nella casella di testo corrispondente aggiungerlo con l'apposito bottone Aggiungi. Il seed apparirà quindi nella lista degli URI Seed inseriti.

Esecuzione di un analisi

L'esecuzione di un analisi prevede che una sessione di analisi sia stata creata o aperta da un precedente archivio. Inoltre prevede l'inserimento di URI Seed da cui far partire l'esplorazione. Fatti questi passi è possibile avviare l'esplorazione con l'apposito tasto Avvia (nel caso di apertura di un archivio con URI ancora da scaricare l'avvio sarà automatico). Il tasto si trasformerà quindi nel tasto per mettere in pausa l'esplorazione Sospendi. Se l'esplorazione è sospesa premere nuovamente Avvia per riprenderla. Nel caso in cui si vuole invece fermare definitivamente l'esecuzione basterà premere il tasto Ferma. Sarà ancora possibile vedere le informazioni relative all'analisi ma non sarà possibile riprenderla o aggiungere nuovi URI Seed da cui far ripartire l'analisi. Basterà infine chiudere la scheda relativa ad un dominio per chiudere la sessione relativa al dominio. Nel caso in cui la sessione chiusa è stata archiviata sarà possibile riprendere l'analisi aprendo l'archivio.

Informazioni sugli URI analizzati

Una volta avviata l'analisi, la tabella relativa agli URI analizzati verrà popolata. Sarà possibile vedere l'URI analizzato e il suo stato di download. Premendo sulla riga relativa all'URI analizzato, nel caso di scaricamento avvenuto con successo, sarà possibile visualizzare informazioni sull'URI. Tali informazioni includono:

  • Visualizzazione della pagina
  • Appartenenza al dominio
  • Numero di immagini presenti nella pagina
  • Numero di nodi dell'albero di parsing
  • Link contenuti nella pagina e relativo numero
  • Link che puntano a quella pagina (link entranti) e relativo numero
La presenza delle informazioni varia in base all'appartenenza dell'URI al dominio specificato.

Nota

La tabella relativa agli URI analizzati lascerà selezionato l'ultimo URI di cui si sono viste le informazioni. Questo per agevolare la leggibilità della tabella. Nel caso si vogliano rivedere le informazioni di quest'ultimo URI si dovrà deselezionare la riga con la combinazione Ctrl+click sinistro (sulla riga selezionata).
Sotto la tabella relativa agli URI analizzati è possibile conoscere lo stato dell'analisi, il numero di pagine processate, cioè analizzate con successo o che hanno ritornato errore e le pagine in coda per l'analisi. Le pagine in coda non corrispondono alle pagine rimanenti da scaricare, ma è un indice dello stato di attività dell'analisi. Quando tale indice raggiunge lo 0 le pagine previste sono state analizzate e le ultime pagine scaricate (se esistono) compariranno nella tabella.

Statistiche di un sito

Vi è la possibilità di vedere informazioni statistiche relative ad un sito. Premere il tasto Strumenti quindi il tasto Statistiche. Saranno mostrate le seguenti informazioni:

    Numero totale URI visitati senza errori
    Il numero di pagine interne ed esterne scaricate con successo.

    Numero URI interni al dominio
    Il numero di pagine interne analizzate con o senza errori

    Numero URI che hanno prodotto errori
    Il numero di pagine interne ed esterne che hanno presentato errori

    Massimo numero di link in una pagina
    Il numero massimo di link contenuti in una pagina scaricata

    Numero massimo di link che puntano ad una pagina
    Il massimo numero di link entranti in una pagina

    Grafico sulle tipologie di URI
    Tale grafico permette di comprendere le tipologie di URI analizzati. Utile per capire la bontà dell'analisi.

Calcolo di link da un sito aperto verso un altro sito aperto

Questa funzionalità permette di capire quanti link uscenti da un sito aperto finiscono ad un altro sito aperto. In altre parole parole quali link entranti del secondo sito sono del primo sito.

Per accedere allo strumento premere il tasto Strumenti dalla scheda relativa al sito sorgente e quindi Link da questo sito verso..

Dal menù a tendina scegliere il sito di destinazione. Si può anche selezionare lo stesso sito sorgente.

Il numero di link corrispondente sarà dunque mostrato.
